HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • h1-h6 represent the level of heading a given text block represents
  • p element represents a paragraph or block of text
  • ul, ol, and li elements represent unordered lists, ordered lists, and list items
  • img element contains information about images that are displayed on the webpage
  • br element creates a line break ebtween two blocks of content
  • a element (which stands for anchor) creates links to a webpage. the href attribute points to the url for the link

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • External CSS style sheet is preferred in most cases because it allows developers to keep all CSS rules in a separate file
  • * is a universal selector (selects all elements on page)
  • Do Not Repeat Yourself (DRY) is a coding principle that encourages developers to reuse or share pieces of code in roder to reduce the number of lines of code that need to be written or maintained

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin main: pulls latest code from main branch and stores on local repository, ensuring your code is up to date with original
  • good practice to create new branches for new features and starting names with "feature/"
  • git push origin (branch name) pushes feature branch to GitHub repository
  • pull request (PR) allows us to merge comparison branch (feature branch) into base branch -- main
  • GitHub workflow: ensuring code is up to date by using "git pull origin main", creating feature branches for new features, pushing features to repo with "git push origin (branch name)", merging feature branch to main by creating and approving a pull request (PR), then ensuring local repo is up to date by using "git pull origin main"
  • version control allows us to store copies/versions of the code at different points which helps undo changes and go back to previous iterations of the code
  • Start ssh-agent using "eval "$(ssh-agent -s)"; add private SSH key using ssh-add

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Console.log prints messages to console -- can be viewed using Chrome DevTools (right click -> inspect)
  • Control flow can be interrupted by using conditional statements and loops to control the order in which code is executed or if it is even executed at all
  • A function is a set of instructions that tells the computer how to perform a certain task
  • Functions are not automatically executed; must be called by name in order to execute
